Git学习笔记

Veröffentlicht am 2017-06-24

注：以下内容建立在本地已经安装好git

首先创建远端仓库,比如在github上.然后打开git命令行，输入：

git clone https://github.com/xxx/xxx.git

设置git用户名、邮箱

git config --global user.name "xxx"
git config --global user.email "xxx@xxx.xxx"

查看git状态：

git status

存入暂存区:

git add "filename"

提交：

git commit

此时会弹出文本编辑器,要求输入提交描述,输入完保存退出即可,另可以在提交的时候使用-m指令直接输入描述:

git commit -m "xxx"

也可以将存入暂存区add步骤省略,实际上是做了缓存只是通过指令-a省略输入add步骤：

git commit -a -m "xxx"

对比命令:

git diff // 对比工作区与缓存区的差异
git diff --cached // 对比缓存区与版本区的差异也可以使用--staged效果与--cached一样
git diff master // 对比工作区与版本库的差异

撤销:

git reset HEAD <filename> // 撤销缓存区
git checkout -- <filename> // 迁出版本库的版本(如果缓存区有版本则还原缓存区的版本)覆盖工作区版本
git commit --amend // 撤销上次提交并与暂存区的内容重新一起提交

删除:

git rm <filename> // 如果工作区文件已删除,则可以使用此命令删除暂存区文件
git rm -f <filename> // 同时删除工作区和暂存区文件
git rm -cached <filename> // 删除暂存区文件

恢复:

git checkout <commitid> <filename> // 从版本库恢复指定版本的指定文件
git reset --hard <commitid> // 恢复指定版本
git reset --hard HEAD^ // 恢复到上一个版本
git reset --hard HEAD~3 // 恢复到前面第三个版本

同步到远程仓库:

git remote // 查看远程仓库
git push origin master // 提交到指定仓库的指定分支

解决冲突:

git fetch // 拉取远端版本不合并
    git diff master origin/master // 比本地master和远端master区别
    git merge origin/master // 合并冲突,然后再通过commit提交,再push到远端
git pull // 直接拉取远端覆盖本地

分支操作:

git branch // 查看分支
git branch <branchname> // 创建分支
git branch -d <branchname> // 删除分支
git branch --merged // 查看已合并分支
git branch --no-merged // 查看未合并分支
git checkout <branchname> // 切换分支
git checkout -b <branchname> // 创建并切换分支
